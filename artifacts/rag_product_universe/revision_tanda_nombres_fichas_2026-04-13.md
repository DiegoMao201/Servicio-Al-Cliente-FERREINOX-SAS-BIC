Fecha: 2026-04-13

Criterio usado:

- Mantener familia ERP estable.
- Corregir typos.
- Quitar nombres demasiado genéricos.
- Separar ficha técnica, FDS, brochure, FAQ y certificado.

## Convención final que te recomiendo usar

### Para fichas técnicas principales

`FAMILIA_ERP_ESTABLE.pdf`

Ejemplos:

- `INTERCHAR 2060.pdf`
- `KORAZA SOL Y LLUVIA IMPERMEABILIZANTE.pdf`
- `VINILTEX ULTRALAVABLE.pdf`
- `PINTULUX 3 EN 1.pdf`
- `THINNER UNIVERSAL P502 MEG AC.pdf`

### Para FDS

`FDS - FAMILIA_VARIANTE_CLAVE.pdf`

Ejemplos:

- `FDS - INTERSEAL 670 HS RAL 5002 PARTE A.pdf`
- `FDS - ACRILICA MANTENIMIENTO 13795 VERDE RAL6025.pdf`

### Para FAQ

`FAQ - FAMILIA.pdf`

Ejemplo:

- `FAQ - VINILTEX ULTRALAVABLE.pdf`

### Para brochures

`BROCHURE - FAMILIA.pdf`

### Para certificaciones

`CERTIFICACION - FAMILIA.pdf`

## Regla de decisión rápida

- Si el nombre describe una familia comercial real, sí.
- Si el nombre solo describe el tipo de documento, no.
- Si el nombre tiene typo, corrígelo primero.
- Si el nombre es demasiado amplio, bájalo a subfamilia.
- Si el archivo es FDS, FAQ, brochure o certificado, no lo mezcles con la ficha técnica principal.
